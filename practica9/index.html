Carro bluetooth

Juan Pablo Sotres Camacho
Ingeniería Mecatrónica
Noviembre 26, 2024
Arduino Electrónica Bluetooth

- Resumen -

En esta práctica aprendimos a como diseñar, programar, ensamblar mecánicamente y conectar electrónicamente un carro y una brazo robótico, y controlarlos vía bluetooth en un aplicación hecha por nosotros.

- Introducción -

En esta práctica, trabajamos en el diseño y desarrollo de dos dispositivos: un carro robótico y un brazo robótico. Aprendimos a ensamblar las partes mecánicas, conectar los componentes electrónicos y programar ambos sistemas para que pudieran moverse y realizar tareas de forma autónoma. Lo más interesante fue que diseñamos una aplicación propia para controlarlos, utilizando Bluetooth para la comunicación entre los dispositivos. Esta práctica nos permitió ver cómo se integran la electrónica, la programación y el diseño mecánico en proyectos de robótica. Este proyecto con el fin de reunir los carros de cada equipo y ponerlos a competir en una arena donde salía mejor beneficiado en la calificación el equipo que recolecte el mayor número de pelotas.

- Desarrollo -

Durante el desarrollo de esta práctica, comenzamos con el diseño y la construcción del carro y el brazo robótico. En la parte mecánica, nos encargamos de seleccionar los materiales adecuados para cada componente, asegurándonos de que fueran lo suficientemente resistentes y ligeros para que los dispositivos pudieran moverse de manera eficiente. En el caso del carro, incorporamos ruedas motorizadas que le permitieron desplazarse en distintas direcciones, mientras que el brazo robótico fue diseñado para realizar movimientos precisos y controlados, imitando el comportamiento de un brazo humano.

Una vez que tuvimos las piezas mecánicas listas, pasamos a la parte electrónica. Aquí, conectamos los motores, sensores y controladores al Arduino y a un Puente H para 4 motores. Para poder controlar tanto el carro como el brazo robótico de forma remota, utilizamos módulos Bluetooth, lo que nos permitió establecer comunicación inalámbrica entre los dispositivos y nuestra aplicación. Estos módulos fueron clave para transmitir las señales de control sin necesidad de cables, facilitando la interacción en tiempo real.

La programación fue otra parte crucial del proceso. Para ello, utilizamos un lenguaje de programación para el Arduino que nos permitió escribir los algoritmos necesarios para que los motores y servomotores respondieran de forma adecuada a las órdenes enviadas desde la aplicación. Además, tuvimos que ajustar el software para garantizar que las señales de control llegaran de manera eficiente y sin latencia significativa, asegurando que tanto el carro como el brazo robótico pudieran ejecutar las tareas que les asignamos.

Usamos una aplicación llamada "MIT AppInventor" que nos permitió crear una interfaz sencilla y fácil de usar, desde donde podíamos controlar los movimientos del carro y del brazo. La interfaz incluía botones para mover el carro en diferentes direcciones y para accionar el brazo robótico, y para conectar y desconectar el dipositivo al modulo bluetooth, lo que le dio a la práctica una dimensión más interactiva. La comunicación entre la aplicación y los dispositivos robóticos se gestionó mediante el protocolo Bluetooth, lo que nos permitió realizar los controles de manera remota y en tiempo real.
